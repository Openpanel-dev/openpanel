為你的 Vue 應用程式加入分析，能幫助你了解使用者如何與你的應用互動。OpenPanel 的 Web SDK 可與 Vue 3 和 Vue Router 順暢整合，提供自動頁面瀏覽追蹤、自訂事件，以及 [使用者識別](/features/identify-users)，而不需要任何 Vue 專屬綁定。

OpenPanel 是一個開源的 Mixpanel 與 Google Analytics 替代方案。它在預設啟用無 cookie 追蹤的同時，也提供強大的洞察，並尊重使用者隱私。

## 必要條件

- 一個 Vue 3 專案（Vite 或 Vue CLI）
- 一個 OpenPanel 帳號（[免費註冊](https://dashboard.openpanel.dev/onboarding)）
- 你在 OpenPanel 儀表板中的 Client ID

## 安裝 SDK [#install]

OpenPanel Web SDK 是一個輕量套件，可在任何 JavaScript 環境中運作，包括 Vue。使用 npm 安裝即可，pnpm 或 yarn 的用法也相同。

```bash
npm install @openpanel/web
```

## 建立 OpenPanel 實例 [#setup]

為你的 OpenPanel 實例建立一個獨立檔案。這樣可以集中管理你的分析設定，也讓這個實例能夠輕鬆地在整個應用程式中匯入使用。

```ts title="src/lib/op.ts"
import { OpenPanel } from '@openpanel/web';

export const op = new OpenPanel({
  clientId: 'YOUR_CLIENT_ID',
  trackScreenViews: true,
  trackOutgoingLinks: true,
  trackAttributes: true,
});
```

`trackScreenViews` 選項會在 URL 變更時自動追蹤頁面瀏覽。這適用於 Vue Router 的前端路由導覽。`trackAttributes` 選項可啟用宣告式追蹤，使用 HTML 元素上的 `data-track` 屬性。

### 使用環境變數

在正式環境應用程式中，請將你的 Client ID 儲存在環境變數中。

```ts title="src/lib/op.ts"
import { OpenPanel } from '@openpanel/web';

export const op = new OpenPanel({
  clientId: import.meta.env.VITE_OPENPANEL_CLIENT_ID,
  trackScreenViews: true,
  trackOutgoingLinks: true,
  trackAttributes: true,
});
```

```bash title=".env"
VITE_OPENPANEL_CLIENT_ID=your-client-id
```

### 在應用程式載入時初始化

在應用程式的進入點匯入 OpenPanel 實例，確保應用程式載入時就會初始化。只要這樣做，你就能開始自動追蹤頁面瀏覽。

```ts title="src/main.ts"
import { createApp } from 'vue';
import App from './App.vue';
import router from './router';
import './lib/op'; // 初始化 OpenPanel

const app = createApp(App);
app.use(router);
app.mount('#app');
```

### 讓 OpenPanel 全域可用（選用）

如果你偏好透過 Vue 實例存取 OpenPanel，而不是在每個元件中匯入它，你可以將它加入為全域屬性。

```ts title="src/main.ts"
import { createApp } from 'vue';
import App from './App.vue';
import router from './router';
import { op } from './lib/op';

const app = createApp(App);
app.config.globalProperties.$op = op;
app.use(router);
app.mount('#app');
```

這樣就能在 Options API 元件中使用 `this.$op`，也可以在 Composition API 元件中注入它。

## 追蹤頁面瀏覽 [#pageviews]

啟用 `trackScreenViews: true` 後，當瀏覽器的 URL 變更時，OpenPanel 會自動追蹤頁面瀏覽。這可直接與 Vue Router 搭配使用。

如果你需要手動追蹤頁面瀏覽，或想加入額外的路由中繼資料，可以使用 Vue Router 的導覽守衛。

```ts title="src/router/index.ts"
import { createRouter, createWebHistory } from 'vue-router';
import { op } from '../lib/op';
import routes from './routes';

const router = createRouter({
  history: createWebHistory(),
  routes,
});

router.afterEach((to) => {
  op.track('screen_view', {
    path: to.path,
    name: String(to.name),
  });
});

export default router;
```

如果你使用這種方式，請在 OpenPanel 設定中將 `trackScreenViews: false`，以避免重複追蹤。

## 追蹤自訂事件 [#events]

在元件中匯入 OpenPanel 實例以追蹤事件。`track` 方法接受事件名稱與可選的屬性物件。

### 使用 Composition API

```vue title="src/components/SignupButton.vue"
<template>
  <button type="button" @click="handleClick">註冊</button>
</template>

<script setup lang="ts">
import { op } from '../lib/op';

function handleClick() {
  op.track('button_clicked', {
    button_name: 'signup',
    button_location: 'hero',
  });
}
</script>
```

### 使用 Options API

```vue title="src/components/SignupButton.vue"
<template>
  <button type="button" @click="handleClick">註冊</button>
</template>

<script>
import { op } from '../lib/op';

export default {
  methods: {
    handleClick() {
      op.track('button_clicked', {
        button_name: 'signup',
        button_location: 'hero',
      });
    },
  },
};
</script>
```

### 追蹤表單送出

表單追蹤可幫助你了解 [轉換率](/features/conversion)，並找出使用者在哪裡流失。

```vue title="src/components/ContactForm.vue"
<template>
  <form @submit.prevent="handleSubmit">
    <input
      v-model="email"
      type="email"
      placeholder="輸入你的電子郵件"
    />
    <button type="submit">送出</button>
  </form>
</template>

<script setup lang="ts">
import { ref } from 'vue';
import { op } from '../lib/op';

const email = ref('');

function handleSubmit() {
  op.track('form_submitted', {
    form_name: 'contact',
    form_location: 'homepage',
  });

  // 你的表單送出邏輯
}
</script>
```

### 使用資料屬性進行宣告式追蹤

Web SDK 支援使用 `data-track` 屬性進行宣告式追蹤。這適合不想寫 JavaScript 的簡單點擊追蹤。

```vue
<template>
  <button
    data-track="button_clicked"
    data-track-button_name="signup"
    data-track-button_location="hero"
  >
    註冊
  </button>
</template>
```

當使用者點擊這個按鈕時，OpenPanel 會自動送出一個帶有指定屬性的 `button_clicked` 事件。這需要你在設定中啟用 `trackAttributes: true`。

## 識別使用者 [#identify]

當使用者登入或提供識別資訊後，呼叫 `identify` 以將其活動與個人資料關聯。這可啟用使用者層級分析與分群分析。

```vue title="src/components/UserProfile.vue"
<template>
  <div>歡迎，{{ user?.firstName }}！</div>
</template>

<script setup lang="ts">
import { watch } from 'vue';
import { op } from '../lib/op';

interface User {
  id: string;
  firstName: string;
  lastName: string;
  email: string;
  plan: string;
  createdAt: string;
}

const props = defineProps<{ user: User | null }>();

watch(
  () => props.user,
  (user) => {
    if (user) {
      op.identify({
        profileId: user.id,
        firstName: user.firstName,
        lastName: user.lastName,
        email: user.email,
        properties: {
          plan: user.plan,
          signupDate: user.createdAt,
        },
      });
    }
  },
  { immediate: true }
);
</script>
```

### 登出時清除使用者資料

當使用者登出時，請清除已儲存的個人資料，以確保後續事件不會與先前的使用者關聯。

```vue title="src/components/LogoutButton.vue"
<template>
  <button @click="handleLogout">登出</button>
</template>

<script setup lang="ts">
import { op } from '../lib/op';

const emit = defineEmits<{ logout: [] }>();

function handleLogout() {
  op.clear();
  emit('logout');
}
</script>
```

### 設定全域屬性

使用 `setGlobalProperties` 設定的屬性會包含在每個事件中。這對於應用程式版本追蹤、功能旗標或 A/B 測試變體很有用。

```vue title="src/App.vue"
<script setup lang="ts">
import { onMounted } from 'vue';
import { op } from './lib/op';

onMounted(() => {
  op.setGlobalProperties({
    app_version: '1.0.0',
    environment: import.meta.env.MODE,
  });
});
</script>
```

## 驗證你的設定 [#verify]

在瀏覽器中開啟你的 Vue 應用程式，並在幾個頁面之間切換。與會觸發自訂事件的元素互動。接著開啟你的 [OpenPanel 儀表板](https://dashboard.openpanel.dev)，查看即時檢視以確認事件是否出現。

如果事件沒有出現，請檢查瀏覽器主控台是否有錯誤。確認你的 Client ID 正確，並確保廣告阻擋工具沒有封鎖對 OpenPanel API 的請求。瀏覽器開發者工具中的 Network 分頁可幫助你確認請求是否已送出。

Web SDK 還有其他功能，例如屬性遞增與事件篩選。請閱讀完整的 [Web SDK 文件](/docs/sdks/web) 以查看完整 API 參考。

對於 Nuxt.js 應用程式，設定方式類似。在 Nuxt 外掛中初始化 OpenPanel，自動頁面瀏覽追蹤就會與 Nuxt 的路由器一起運作。

<Faqs>
<FaqItem question="OpenPanel 可以和 Vue Router 一起使用嗎？">
可以。OpenPanel 會在 URL 變更時自動追蹤頁面瀏覽，這可與 Vue Router 搭配使用。請在設定中將 `trackScreenViews: true`。你也可以使用路由導覽守衛進行手動追蹤。
</FaqItem>

<FaqItem question="我可以在 Vue 2 中使用 OpenPanel 嗎？">
可以。OpenPanel Web SDK 不依賴特定框架，且可與 Vue 2 搭配使用。直接在元件中匯入實例並呼叫其方法即可。
</FaqItem>

<FaqItem question="OpenPanel 會使用 cookies 嗎？">
不會。OpenPanel 預設使用無 cookie 追蹤。這表示在大多數隱私法規下，包括 GDPR 與 PECR，你不需要為基本分析顯示 cookie 同意橫幅。
</FaqItem>

<FaqItem question="OpenPanel 符合 GDPR 規範嗎？">
是的。OpenPanel 以無 cookie 追蹤、資料最小化，以及完整支援資料主體權利為設計目標，符合 GDPR 規範。使用自架部署時，你也能完全消除國際資料傳輸的疑慮。
</FaqItem>
</Faqs>
