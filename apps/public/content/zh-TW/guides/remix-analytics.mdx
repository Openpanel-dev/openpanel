為您的 Remix 應用程式加入分析追蹤，有助於您了解使用者如何與應用程式互動。OpenPanel 的 Web SDK 可與 Remix 的用戶端導覽無縫搭配，提供自動頁面瀏覽追蹤、自訂事件，以及 [使用者識別](/features/identify-users)。

OpenPanel 是 Mixpanel 和 Google Analytics 的開源替代方案。它在預設使用無 Cookie 追蹤的同時，仍提供強大的洞察，並尊重使用者隱私。

## 先決條件

- 一個 Remix 專案
- 一個 OpenPanel 帳號（[免費註冊](https://dashboard.openpanel.dev/onboarding)）
- 您在 OpenPanel 儀表板中的 Client ID

## 安裝 SDK [#install]

OpenPanel Web SDK 是一個輕量套件，可在任何 JavaScript 環境中使用。使用 npm 安裝；pnpm 或 yarn 的用法也相同。

```bash
npm install @openpanel/web
```

## 建立 OpenPanel 實例 [#setup]

為您的 OpenPanel 實例建立一個專用檔案。由於這會在瀏覽器中執行，請將它放在您的 app 目錄中，並確保它只在用戶端執行。

```ts title="app/lib/op.client.ts"
import { OpenPanel } from '@openpanel/web';

export const op = new OpenPanel({
  clientId: 'YOUR_CLIENT_ID',
  trackScreenViews: true,
  trackOutgoingLinks: true,
  trackAttributes: true,
});
```

`.client.ts` 副檔名會告訴 Remix 這個模組只應在瀏覽器中執行。`trackScreenViews` 選項會在 URL 變更時自動追蹤頁面瀏覽，這可與 Remix 的用戶端導覽搭配使用。`trackAttributes` 選項則啟用使用 `data-track` 屬性的宣告式追蹤。

### 使用環境變數

對於正式環境應用程式，請使用 Remix 的 loader 模式，將 Client ID 從伺服器傳遞到用戶端。

```ts title="app/root.tsx"
import { json } from '@remix-run/node';
import type { LoaderFunctionArgs } from '@remix-run/node';

export async function loader({ request }: LoaderFunctionArgs) {
  return json({
    ENV: {
      OPENPANEL_CLIENT_ID: process.env.OPENPANEL_CLIENT_ID,
    },
  });
}
```

接著在用戶端元件中，使用環境變數初始化 OpenPanel。

### 在 root.tsx 中初始化

在您的 root 元件中匯入並初始化 OpenPanel，並使用 `useEffect` hook 確保它只在用戶端執行。

```tsx title="app/root.tsx"
import { useEffect } from 'react';
import {
  Links,
  Meta,
  Outlet,
  Scripts,
  ScrollRestoration,
  useLoaderData,
} from '@remix-run/react';

export default function App() {
  const { ENV } = useLoaderData<typeof loader>();

  useEffect(() => {
    // 動態匯入可確保這只會在用戶端執行
    import('./lib/op.client').then(({ op }) => {
      // OpenPanel 現在已初始化並開始追蹤
    });
  }, []);

  return (
    <html lang="en">
      <head>
        <meta charSet="utf-8" />
        <meta name="viewport" content="width=device-width, initial-scale=1" />
        <Meta />
        <Links />
      </head>
      <body>
        <Outlet />
        <ScrollRestoration />
        <Scripts />
      </body>
    </html>
  );
}
```

## 追蹤頁面瀏覽 [#pageviews]

啟用 `trackScreenViews: true` 後，當瀏覽器的 URL 變更時，OpenPanel 會自動追蹤頁面瀏覽。這可與 Remix 使用 `<Link>` 元件的用戶端導覽搭配運作。

如果您需要對頁面瀏覽追蹤有更多控制，或想加入額外的路由中繼資料，可以建立一個使用 Remix `useLocation` hook 的元件。

```tsx title="app/components/PageTracker.tsx"
import { useEffect } from 'react';
import { useLocation } from '@remix-run/react';

export function PageTracker() {
  const location = useLocation();

  useEffect(() => {
    import('../lib/op.client').then(({ op }) => {
      op.track('screen_view', {
        path: location.pathname,
        search: location.search,
      });
    });
  }, [location.pathname, location.search]);

  return null;
}
```

將這個元件加入您的 root 版面配置中。如果您使用這種方式，請在 OpenPanel 設定中將 `trackScreenViews: false`，以避免重複追蹤。

## 追蹤自訂事件 [#events]

在您的元件中匯入 OpenPanel 實例以追蹤事件。由於 SDK 只能在瀏覽器中運作，請使用動態匯入，或確保您的追蹤程式碼在 `useEffect` hooks 中執行。

```tsx title="app/components/SignupButton.tsx"
export function SignupButton() {
  const handleClick = () => {
    import('../lib/op.client').then(({ op }) => {
      op.track('button_clicked', {
        button_name: 'signup',
        button_location: 'hero',
      });
    });
  };

  return (
    <button type="button" onClick={handleClick}>
      註冊
    </button>
  );
}
```

### 建立追蹤 hook

為了讓程式碼更簡潔，您可以建立一個自訂 hook 來處理動態匯入。

```tsx title="app/hooks/useOpenPanel.ts"
import { useCallback } from 'react';
import type { OpenPanel } from '@openpanel/web';

type TrackFn = OpenPanel['track'];
type IdentifyFn = OpenPanel['identify'];

export function useTrack() {
  return useCallback<TrackFn>((name, properties) => {
    import('../lib/op.client').then(({ op }) => {
      op.track(name, properties);
    });
  }, []);
}

export function useIdentify() {
  return useCallback<IdentifyFn>((payload) => {
    import('../lib/op.client').then(({ op }) => {
      op.identify(payload);
    });
  }, []);
}
```

現在您的元件會更簡潔。

```tsx title="app/components/SignupButton.tsx"
import { useTrack } from '../hooks/useOpenPanel';

export function SignupButton() {
  const track = useTrack();

  const handleClick = () => {
    track('button_clicked', {
      button_name: 'signup',
      button_location: 'hero',
    });
  };

  return (
    <button type="button" onClick={handleClick}>
      註冊
    </button>
  );
}
```

### 追蹤表單提交

Remix 鼓勵使用 form actions 進行資料異動。您可以在 action 處理器中，或在用戶端追蹤表單提交。

```tsx title="app/routes/contact.tsx"
import { Form } from '@remix-run/react';
import { useTrack } from '../hooks/useOpenPanel';

export default function Contact() {
  const track = useTrack();

  const handleSubmit = () => {
    track('form_submitted', {
      form_name: 'contact',
      form_location: 'contact-page',
    });
  };

  return (
    <Form method="post" onSubmit={handleSubmit}>
      <input type="email" name="email" placeholder="您的電子郵件" required />
      <button type="submit">送出</button>
    </Form>
  );
}
```

### 使用資料屬性進行宣告式追蹤

Web SDK 支援使用 `data-track` 屬性的宣告式追蹤。這對於不需撰寫 JavaScript 的簡單點擊追蹤非常有用。

```tsx
<button
  data-track="button_clicked"
  data-track-button_name="signup"
  data-track-button_location="hero"
>
  註冊
</button>
```

當使用者點擊這個按鈕時，OpenPanel 會自動送出一個帶有指定屬性的 `button_clicked` 事件。這需要您在設定中啟用 `trackAttributes: true`。

## 識別使用者 [#identify]

當使用者登入後，請呼叫 `identify` 以將其活動與個人資料關聯起來。在 Remix 中，您通常可以從 loader 取得使用者資料。

```tsx title="app/routes/dashboard.tsx"
import { useEffect } from 'react';
import { useLoaderData } from '@remix-run/react';
import type { LoaderFunctionArgs } from '@remix-run/node';
import { json } from '@remix-run/node';
import { getUser } from '../lib/auth.server';

export async function loader({ request }: LoaderFunctionArgs) {
  const user = await getUser(request);
  return json({ user });
}

export default function Dashboard() {
  const { user } = useLoaderData<typeof loader>();

  useEffect(() => {
    if (user) {
      import('../lib/op.client').then(({ op }) => {
        op.identify({
          profileId: user.id,
          firstName: user.firstName,
          lastName: user.lastName,
          email: user.email,
          properties: {
            plan: user.plan,
          },
        });
      });
    }
  }, [user]);

  return <div>歡迎，{user?.firstName}！</div>;
}
```

### 登出時清除使用者資料

當使用者登出時，請清除已儲存的個人資料，以確保後續事件不會與先前的使用者關聯。

```tsx title="app/components/LogoutButton.tsx"
import { Form } from '@remix-run/react';

export function LogoutButton() {
  const handleClick = () => {
    import('../lib/op.client').then(({ op }) => {
      op.clear();
    });
  };

  return (
    <Form method="post" action="/logout">
      <button type="submit" onClick={handleClick}>
        登出
      </button>
    </Form>
  );
}
```

## 伺服器端追蹤 [#server-side]

若要在 loader、action 或 API 路由中追蹤事件，請改用 `@openpanel/sdk` 套件，而不是 Web SDK。伺服器端追蹤需要 client secret。

```ts title="app/lib/op.server.ts"
import { OpenPanel } from '@openpanel/sdk';

export const op = new OpenPanel({
  clientId: process.env.OPENPANEL_CLIENT_ID!,
  clientSecret: process.env.OPENPANEL_CLIENT_SECRET!,
});
```

```ts title="app/routes/api.webhook.ts"
import type { ActionFunctionArgs } from '@remix-run/node';
import { json } from '@remix-run/node';
import { op } from '../lib/op.server';

export async function action({ request }: ActionFunctionArgs) {
  const payload = await request.json();

  op.track('webhook_received', {
    source: payload.source,
    event_type: payload.type,
  });

  return json({ success: true });
}
```

## 驗證您的設定 [#verify]

在瀏覽器中開啟您的 Remix 應用程式，並在幾個頁面之間切換。與會觸發自訂事件的元素互動。然後開啟您的 [OpenPanel 儀表板](https://dashboard.openpanel.dev)，查看即時檢視，確認事件是否出現。

如果事件沒有出現，請檢查瀏覽器主控台是否有錯誤。確認您的 Client ID 正確，並確保廣告攔截器沒有封鎖對 OpenPanel API 的請求。瀏覽器開發者工具中的 Network 分頁可協助您確認請求是否已送出。

## 後續步驟

Web SDK 還有其他功能，例如屬性遞增與事件篩選。請閱讀完整的 [Web SDK 文件](/docs/sdks/web) 以取得完整 API 參考。

若要進行完整的伺服器端追蹤，請參閱 [Node.js 分析指南](/guides/nodejs-analytics)，其中詳細說明了 `@openpanel/sdk` 套件。

<Faqs>
<FaqItem question="OpenPanel 可以搭配 Remix 的 SSR 使用嗎？">
可以。OpenPanel 的用戶端 SDK 會在 hydration 後於瀏覽器中追蹤事件。若要在 loader 和 action 中追蹤伺服器端事件，請使用 `@openpanel/sdk` 套件並提供您的 client secret。
</FaqItem>

<FaqItem question="為什麼伺服器端追蹤需要 client secret？">
伺服器端追蹤需要驗證，因為我們無法使用 CORS 標頭。client secret 可確保您的事件經過正確驗證，並防止未經授權的追蹤。
</FaqItem>

<FaqItem question="OpenPanel 會使用 Cookie 嗎？">
不會。OpenPanel 預設使用無 Cookie 追蹤。這表示在大多數隱私法規下，包括 GDPR 與 PECR，您不需要為基本分析顯示 Cookie 同意橫幅。
</FaqItem>

<FaqItem question="OpenPanel 符合 GDPR 嗎？">
是。OpenPanel 以符合 GDPR 為設計目標，具備無 Cookie 追蹤、資料最小化，以及對資料當事人權利的完整支援。若採用自架部署，您也能完全消除國際資料傳輸的疑慮。
</FaqItem>
</Faqs>
