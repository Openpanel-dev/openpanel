[Dokploy](https://dokploy.com) 是一個開源、自架的平台，用於部署應用程式。OpenPanel 可透過 Docker Compose 範本部署在 Dokploy 上，但需要特定的設定。

<Callout type="warn">
⚠️ **重要**：Dokploy 範本需要與 Coolify 不同的特定設定。請務必仔細完成所有步驟，尤其是環境變數與網域設定。
</Callout>
<Callout type="warn">
⚠️ **重要**：我們在 dokploy 上有一個開放問題 https://github.com/Dokploy/templates/issues/292，並希望它能被解決。
</Callout>

## 先決條件

- 已安裝並執行中的 Dokploy 實例
- 至少 2GB RAM 的伺服器（建議 4GB 以上）
- 已在 Dokploy 中設定網域名稱

## 快速開始

<Steps>
<Step>
### 部署 OpenPanel 範本

1. 登入你的 Dokploy 儀表板
2. 前往你的專案
3. 點選 **"New Application"** 或 **"Deploy"**
4. 選擇 **"Docker Compose"**，或搜尋 **"OpenPanel"** 範本
5. 選取 OpenPanel 範本
</Step>

<Step>
### 設定網域名稱

在 Dokploy 中設定你的網域名稱：

1. 為儀表板設定主網域（例如 `analytics.yourdomain.com`）
2. 設定 API 網域 - 這應該與儀表板使用**相同網域**，並將 API 路徑轉送到 `/api`

<Callout>
API 與儀表板使用相同的網域。API 服務的轉送路徑為 `/api`，請務必正確設定。
</Callout>
</Step>

<Step>
### 設定環境變數

編輯 Dokploy 中的 `.env` 檔案或環境變數。你**必須**設定以下環境變數：

```bash
# Required: Set these to your actual domain
API_URL=https://yourdomain.com/api
DASHBOARD_URL=https://yourdomain.com

# Database Configuration (automatically set by Dokploy)
OPENPANEL_POSTGRES_DB=openpanel-db
SERVICE_USER_POSTGRES=postgres
SERVICE_PASSWORD_POSTGRES=<auto-generated>
SERVICE_PASSWORD_REDIS=<auto-generated>

# Optional Configuration
OPENPANEL_ALLOW_REGISTRATION=false
OPENPANEL_ALLOW_INVITATION=true
RESEND_API_KEY=your-resend-api-key
OPENPANEL_EMAIL_SENDER=noreply@yourdomain.com
```

<Callout type="warn">
⚠️ **關鍵**：與 Coolify 不同，Dokploy 不支援 `SERVICE_FQDN_*` 變數。你**必須**以實際網域值硬編碼 `API_URL` 和 `DASHBOARD_URL`。
</Callout>
</Step>

<Step>
### 設定 API 服務網域

在 Dokploy 中，設定 API 服務網域：

1. 前往 `op-api` 服務設定
2. 設定網域配置：
   ```toml
   [[config.domains]]
   serviceName = "op-api"
   port = 3000
   host = "${api_domain}"
   ```
3. **重要**：請勾選 API 服務的 **"Strip external path"** 核取方塊

<Callout>
"Strip external path" 選項非常重要！沒有它，當請求從 `/api` 轉送時，API 會收到錯誤的路徑。
</Callout>
</Step>

<Step>
### 部署

1. 檢查所有設定
2. 點選 **"Deploy"** 或 **"Save"**
3. 等待所有服務啟動並進入健康狀態

監控部署紀錄，確認所有服務都正確啟動。
</Step>

<Step>
### 驗證安裝

部署完成後：

1. 確認所有服務都在執行：
   - `op-api` - API 伺服器
   - `op-dashboard` - 儀表板（前端）
   - `op-worker` - 背景 worker
   - `op-db` - PostgreSQL 資料庫
   - `op-kv` - Redis 快取
   - `op-ch` - ClickHouse 資料庫

2. 以你設定的網域開啟儀表板
3. 嘗試建立帳號，確認 API 運作正常

<Callout type="warn">
如果你在 Dokploy 前面使用 Cloudflare，請記得在變更後清除 Cloudflare 快取，以確保提供最新資源。
</Callout>
</Step>
</Steps>

## 設定細節

### 必要環境變數

對於 Dokploy，你**必須**手動硬編碼以下變數（與 Coolify 不同，Dokploy 不支援 `SERVICE_FQDN_*` 變數）：

- `API_URL` - 完整 API URL（例如 `https://analytics.example.com/api`）
- `DASHBOARD_URL` - 完整儀表板 URL（例如 `https://analytics.example.com`）

Dokploy 會自動設定：
- `OPENPANEL_POSTGRES_DB` - PostgreSQL 資料庫名稱
- `SERVICE_USER_POSTGRES` - PostgreSQL 使用者名稱
- `SERVICE_PASSWORD_POSTGRES` - PostgreSQL 密碼（自動產生）
- `SERVICE_PASSWORD_REDIS` - Redis 密碼（自動產生）

如需所有可用環境變數的完整參考，請參閱 [環境變數文件](/zh-TW/docs/self-hosting/environment-variables)。

### 網域設定

API 與儀表板服務共用相同網域：

- **儀表板**：在根路徑（`/`）提供前端
- **API**：在 `/api` 路徑提供 API

**API 服務的重要設定：**
- 網域：與儀表板網域相同
- 路徑：`/api`
- **Strip external path**：✅ **必須勾選**

這可確保當請求送到 `/api/*` 時，路徑會正確轉送到 API 服務，且不含 `/api` 前綴。

## 疑難排解

### API 請求無法運作

如果部署後 API 請求失敗：

1. **確認環境變數**：
   ```bash
   # Check that API_URL is set correctly
   docker exec <op-api-container> env | grep API_URL
   docker exec <op-dashboard-container> env | grep API_URL
   ```

2. **檢查 "Strip external path" 設定**：
   - 前往 Dokploy 中的 API 服務設定
   - 確認 "Strip external path" 已**勾選**

3. **確認網域設定**：
   - API 服務應使用路徑 `/api`
   - 儀表板服務應位於根目錄 `/`

### 無法建立帳號

如果建立帳號失敗：

1. 檢查 API 紀錄：
   ```bash
   # In Dokploy, view logs for op-api service
   ```

2. 確認 `API_URL` 與你的網域相符：
   - 應為 `https://yourdomain.com/api`
   - 不應是 `http://localhost:3000` 或類似值

3. 檢查 API 服務是否可存取：
   ```bash
   curl https://yourdomain.com/api/healthcheck
   ```

### Cloudflare 快取問題

如果你在 Dokploy 前面使用 Cloudflare：

1. 部署或更新後，請清除 Cloudflare 快取
2. 這可確保立即提供更新後的資源
3. 你可以在 Cloudflare 儀表板或 API 中執行

### 資料庫連線問題

如果服務無法連線到資料庫：

1. 確認資料庫服務正在執行：
   - `op-db`（PostgreSQL）
   - `op-kv`（Redis）
   - `op-ch`（ClickHouse）

2. 檢查環境變數是否已設定：
   ```bash
   docker exec <op-api-container> env | grep DATABASE_URL
   ```

3. 確認 docker-compose 中的服務名稱一致：
   - 資料庫服務名稱：`op-db`、`op-kv`、`op-ch`
   - 這些應與連線字串中的主機名稱相符

## Docker Compose 結構

OpenPanel 範本包含以下服務：

- **op-api**：OpenPanel API 伺服器
- **op-dashboard**：OpenPanel 儀表板（前端）
- **op-worker**：處理事件的背景 worker
- **op-db**：PostgreSQL 資料庫
- **op-kv**：Redis 快取
- **op-ch**：ClickHouse 分析資料庫

## 與 Coolify 的差異

Dokploy 範本與 Coolify 的差異如下：

1. **環境變數**：
   - Dokploy 不支援 `SERVICE_FQDN_*` 變數
   - 必須硬編碼 `API_URL` 和 `DASHBOARD_URL`

2. **網域設定**：
   - 必須手動設定網域路徑
   - 必須為 API 服務啟用 "Strip external path"

3. **服務探索**：
   - 使用標準 Docker Compose 服務名稱
   - 沒有自動 FQDN 解析

## 更新 OpenPanel

要在 Dokploy 中更新 OpenPanel：

1. 拉取最新映像：
   ```bash
   docker compose pull
   ```

2. 重新啟動服務：
   ```bash
   docker compose up -d
   ```

或者使用 Dokploy 的 UI 重新啟動服務。
