## 頂部統計

頁面頂端的指標卡列是最快了解專案健康狀況的方式。每張卡都會顯示所選時間範圍的數值，以及與相同長度前一期間的比較。

### 唯一訪客

在所選期間內記錄到的不同 profile ID 數量。其準確度取決於你是否使用 [identify](/zh-TW/docs/get-started/identify-users)：

- **未使用 identify**：OpenPanel 會產生一個匿名 profile ID，並每 24 小時輪替一次。若某位訪客在 10 天內回訪，會被計為 10 位唯一訪客，因為每天都會產生新的 ID。
- **使用 identify**：profile ID 會綁定到使用者的真實身份。同一個人在 10 天內造訪，整段期間只會計為 1 位唯一訪客。

如果你的分析需要跨日去重，請設定 [使用者識別](/zh-TW/docs/get-started/identify-users)。

### 會話

所選期間內的會話總數。當有人到達你的網站時，會話開始；在 30 分鐘無活動後或關閉分頁時結束。單一訪客在一天內可以有多個會話。

### 瀏覽量

所有會話中記錄到的頁面瀏覽總數（`screen_view` events）。訪客每載入一次頁面——包含在單一會話中於頁面之間切換——都算一次瀏覽量。

### 每次會話頁數

單一會話中平均瀏覽的頁面數，計算方式為 `總瀏覽量 / 總會話數`。數值越高，代表訪客在離開前探索了更多內容。

### 跳出率

訪客只瀏覽單一頁面就離開的會話百分比。計算方式為 `單頁會話 / 總會話數 × 100`。通常越低越好，表示更多訪客不只停留在第一頁。

> 若訪客在會話結束前剛好觸發一次 `screen_view` event，該會話會被計為跳出。即使訪客深入閱讀一篇文章後離開，仍然算作跳出。

### 會話時長

會話平均長度，以秒為單位；僅以訪客在首次載入頁面後有進一步操作的會話（duration > 0）計算。訪客一進入就離開的會話會排除在平均值之外，以避免數值失真。

### 收入

在所選期間內透過 `revenue` events 追蹤的總金額，顯示為你的帳戶幣別。只有在你有追蹤收入事件時才會顯示收入。設定方式請參考 [收入追蹤文件](/zh-TW/features/revenue-tracking)。

## 時間序列圖表

統計卡下方是一張折線圖，顯示所選指標隨時間的變化。點擊任何統計卡即可切換圖表指標。

圖表會使用你選擇的 **interval**（hour、day、week 或 month）來分組資料點。淡色虛線會顯示前一個比較區間的對應期間，讓你一眼看出趨勢。

當目前顯示的指標不是 Revenue 時，圖表也會在第二個 Y 軸上以綠色長條疊加收入，方便你不切換卡片就能將流量模式與收入對照。

折線末端的尾段（目前尚未完整的區間）會以虛線段顯示，提醒你該期間資料仍在累積中。

## 洞察

當你的專案至少有 30 天資料後，圖表下方會出現一排可捲動的洞察卡。OpenPanel 會自動偵測在 pageviews、entry pages、referrers 與 countries 之間的重要趨勢，不需要任何設定。

每張卡會顯示：
- **Share**：該屬性占總流量的百分比（例如：「United States：占全部 sessions 的 42%」）
- **Absolute change**：與前一期間相比，sessions 的實際增加或減少量
- **Percentage change**：相對於該屬性前一個數值的成長或下降百分比

例如，如果上週美國有 1,000 sessions，而本週有 1,200 sessions，卡片會顯示「+200 sessions（+20%）」。

點擊任何洞察卡會將整個總覽頁面篩選為只顯示符合該屬性的資料，讓你進一步深入找出趨勢背後的原因。

## 來源

Sources 小工具會顯示訪客從哪裡來。可切換分頁查看不同維度：

| 分頁 | 顯示內容 |
|-----|---------------|
| **Refs** | 分組後的 referrer 名稱（例如「Google」、「Twitter」、「Hacker News」） |
| **Urls** | 原始 referrer URLs |
| **Types** | Referrer 類別：`search`、`social`、`email`、`unknown` |
| **Source** | `utm_source` 查詢參數值 |
| **Medium** | `utm_medium` 查詢參數值 |
| **Campaign** | `utm_campaign` 查詢參數值 |
| **Term** | `utm_term` 查詢參數值 |
| **Content** | `utm_content` 查詢參數值 |

Referrer 名稱與類型會透過內建對照表，根據原始 referrer URL 自動解析。直接流量（沒有 referrer）會顯示為 `(not set)`。

每一列都會顯示 sessions 與 pageviews。點擊某一列會將整個總覽頁面篩選為只顯示來自該來源的資料。

## 頁面

Pages 小工具會顯示訪客進入、離開，以及停留時間最長的是哪些 URLs。

| 分頁 | 顯示內容 |
|-----|---------------|
| **Top pages** | 依唯一 sessions 排序的頁面。每一列都是 `origin + path` 的組合。 |
| **Entry pages** | 每個會話的第一個頁面，也就是訪客進入時到達的頁面。 |
| **Exit pages** | 每個會話的最後一個頁面，也就是訪客離開時所在的頁面。 |

某個頁面的高退出率不一定是壞事——它可能代表該頁已成功回答了訪客的問題。入口頁面的高跳出率更有診斷價值。比較 entry 與 exit 的分佈，可以了解使用者旅程的形狀。

點擊某個頁面列會將整個總覽篩選為包含該頁面的 sessions。

## 裝置

Devices 小工具會依硬體與軟體拆解你的受眾。可切換分頁：

| 分頁 | 顯示內容 |
|-----|---------------|
| **Device** | 裝置類型：Desktop、Mobile、Tablet |
| **Brand** | 硬體品牌（Apple、Samsung 等） |
| **Model** | 特定裝置型號 |
| **Browser** | 瀏覽器名稱（Chrome、Safari、Firefox 等） |
| **Browser ver.** | 瀏覽器版本號 |
| **OS** | 作業系統（macOS、Windows、iOS、Android 等） |
| **OS ver.** | 作業系統版本 |

每一列都會顯示 sessions 與 pageviews。使用這個小工具可優先決定要測試與最佳化哪些瀏覽器和作業系統。

## 事件

Events 小工具會顯示所選期間內最常觸發的自訂事件，並依次數排序。系統事件（`session_start`、`session_end`、`screen_view`）會被排除——這裡只會顯示你自行埋點的事件。

點擊任何事件即可將總覽篩選為觸發該事件的 sessions。

## 地理位置

Geo 小工具會顯示訪客的地理分佈。可切換分頁：

| 分頁 | 顯示內容 |
|-----|---------------|
| **Country** | 訪客國家/地區，根據 IP 地理定位推算 |
| **Region** | 州或省 |
| **City** | 城市層級 |

表格下方的世界地圖會以熱度圖方式呈現相同資料——顏色越深代表 sessions 越多。這能讓你快速看出受眾集中在哪些地區。

點擊國家、地區或城市會將整個總覽篩選為該位置。

## 活動熱圖

頁面底部的活動熱圖會依星期幾（星期一到星期日）與一天中的時段（00:00–23:00）顯示訪客最活躍的時間。每個格子都會顯示所選指標在該星期幾與小時組合下的 **平均值**，並以所選期間內所有週數平均。

較深的格子表示平均值較高。將滑鼠移到任一格上可查看精確平均值。

你可以使用熱圖上方的分頁切換要視覺化的指標：

- **唯一訪客**
- **會話**
- **瀏覽量**
- **跳出率**
- **每次會話頁數**
- **會話時長**

使用熱圖來找出流量高峰時段、規劃行銷活動，並在離峰時段安排維護。

## 使用者旅程

最底部的 User Journey（Sankey）圖會視覺化訪客在單一會話中如何在你的網站內流動。它回答的是：訪客到達頁面 A 之後，下一步會去哪裡？

**運作方式：**

1. OpenPanel 會找出所選期間內最常見的 3 個入口頁面。
2. 從每個入口頁面開始，找出最常見的 3 個下一頁（第 2 步），再從那些頁面找出各自最常見的 3 個下一頁（第 3 步），依此類推，直到設定的步數上限（預設 5，可調整到最多 10）。
3. 代表少於總 sessions 0.25% 的路徑會被過濾，以減少視覺雜訊。
4. 同一個會話中連續重複的頁面會合併為一步（例如，如果有人重新整理頁面，在旅程中只會計一次）。

每個節點都會顯示頁面 URL。連接流線的寬度與沿著該路徑的 sessions 數量成正比。

使用 User Journey 來找出流失點、發現意外但熱門的路徑，並了解訪客是否有到達你的關鍵轉換頁面。

## 篩選器與時間控制

總覽頁面上的每個小工具都會套用頁面頂端相同的一組全域篩選器與時間控制。

**Range**：選擇預設值（Today、Last 7 days、Last 30 days 等）或自訂日期範圍。

**Interval**：控制時間序列圖表如何分組資料（hour、day、week、month）。

**Event filter**：將整個總覽縮小到包含特定事件的 sessions——適合分析完成某個特定動作的使用者行為。

**Dimension filters**：在任何小工具中點擊任一列（國家、來源、頁面）都會將該值套用為篩選條件。啟用中的篩選器會以 chips 顯示在時間控制下方。點擊 chip 上的 × 即可移除篩選器。

**Live counter**：右上角的綠色徽章會顯示目前活躍訪客數（在過去 5 分鐘內觸發事件的訪客）。點擊可查看 30 分鐘的 session histogram。
