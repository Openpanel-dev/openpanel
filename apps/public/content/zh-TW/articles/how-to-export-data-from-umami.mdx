使用 [網站分析](/features/web-analytics) 時，您會希望能掌控自己的資料。如果您正在更換平台、建立備份，或是想在其他工具中分析資料，就需要一種方式從 Umami Analytics 匯出資料。

Umami 會依據您使用的是雲端服務還是自架環境，採用不同的資料匯出方式。本指南會逐一說明這兩種情況，並解釋為什麼您可能會考慮像 OpenPanel 這樣的替代方案。

## 了解 Umami 的匯出限制

Umami Analytics 是一款不錯、重視隱私的分析工具，但在資料可攜性方面有其限制。雲端版本提供匯出功能，若您有付費使用他們的服務，這相當方便。不過，如果您是自架 Umami，就沒有官方的匯出工具。

之所以有這個落差，是因為 Umami 團隊更專注於分析功能，而不是資料遷移工具。這可以理解，但對於需要移轉資料的使用者來說，這確實是個問題。

許多自架使用者往往在需要遷移或建立備份時，才發現這個限制。您可能有好幾個月甚至好幾年的寶貴分析資料，被鎖在 PostgreSQL 資料庫中，卻沒有簡單的方法把它匯出。

## 雲端匯出功能

如果您使用的是 Umami Cloud，就能使用內建的匯出功能。流程很簡單，但有幾個地方需要注意。

要從 Umami Cloud 匯出，請登入您的帳號並前往 Account 區段。在那裡您會找到 Data 分頁，其中有匯出選項。

提交匯出請求後，Umami 會在背景處理，因此您不必在資料準備期間一直等待。匯出完成後，他們會寄一封含有下載連結的電子郵件給您。

匯出的資料會以 CSV 格式提供，並以 gzip 壓縮以減少檔案大小。如果您經營的是每月有數百萬次瀏覽的高流量網站，這會很有幫助。壓縮通常可將檔案大小減少 70% 到 90%，讓下載更快。

不過還是有一些限制。每次匯出請求一次只能處理一個網站。如果您追蹤多個網域，就必須為每個網域各提交一次匯出請求；如果您管理很多網站，這會變得很繁瑣。

大型資料集的處理時間也可能很長。Umami 不提供即時進度更新，所以您只能等待電子郵件通知。對於非常大型的網站，匯出可能需要數小時才能完成。

## 自架解決方案：OpenPanel 匯出工具

對於自架 Umami 的使用者，我們開發了 [@openpanel/umami-exporter](https://www.npmjs.com/package/@openpanel/umami-exporter) 來補足這個缺口。它會直接連線到您的 PostgreSQL 資料庫，並將所有分析資料匯出為 CSV 格式。

您不需要永久安裝任何東西，也不需要修改您的 Umami 安裝。這個工具可獨立執行，只需要對資料庫的讀取權限。

透過 npx，您可以在不安裝任何東西的情況下執行匯出工具：

```bash
npx op-umami-exporter "postgresql://username:password@host:port/database"
```

這一條命令就能處理所有事情：連線到資料庫、查詢必要的資料表、將資料串接起來，並輸出成可直接匯入其他系統的格式。

這個匯出工具會從多個資料表提取資料，包括 sessions、events 和 website configurations，然後將它們合併成非正規化格式。這讓匯出的資料在其他工具中更容易使用。

一個常見的障礙是，Umami 預設的 Docker 設定不會開放 PostgreSQL 連接埠。您需要修改 docker-compose.yml 檔案來加入連接埠對應。這是一個一次性的變更，只需要一分鐘：

```yaml
services:
  postgres:
    ports:
      - "5432:5432"
```

完成這個變更並重新啟動容器後，匯出工具就能連線到您的資料庫。Umami 的預設憑證通常是使用者名稱與密碼都為 umami/umami，但您還是應該在設定中確認。

## 為什麼要考慮從 Umami 轉換？

Umami 很擅長隱私導向的網站分析，但您可能會需要更進階的功能。這就是像 OpenPanel 這樣的平台能發揮作用的地方。

我們打造 OpenPanel，就是為了滿足不只需要頁面瀏覽追蹤的網站。它保留了與 Umami 相同的隱私優先理念，並加入更多功能，讓您能更深入了解使用者行為。

其中一個差異是即時資料處理。您能即時看到訪客活動，而不是像批次處理系統那樣有延遲。這有助於您快速回應流量高峰、行銷活動或技術問題。

我們的 [事件追蹤](/features/event-tracking) 不只限於頁面瀏覽。您可以追蹤自訂事件、使用者互動，以及 [轉換漏斗](/features/funnels)，而不需要撰寫複雜程式碼。OpenPanel 也會自動擷取許多在 Umami 中原本需要手動設定的互動。

篩選與分群功能也更進階。您可以為特定使用者區隔建立查詢、更彈性地比較不同時間區間，並建立聚焦於您在意指標的自訂儀表板。

對團隊來說，OpenPanel 也有更好的協作功能。您可以分享特定檢視、為利害關係人建立唯讀存取，並設定自動化報表，讓大家都能掌握狀況，而不必給予完整的分析存取權。

## 做出遷移決策

要不要從 Umami 轉移，取決於您的分析需求，以及您預期的成長方式。

如果您經營的是簡單的部落格或作品集網站，Umami 的基本分析功能可能已經足夠。它的簡潔性也能讓您更專注於內容創作。

但如果您經營的是電子商務網站、SaaS 應用程式，或任何使用者行為會直接影響營收的業務，您很快就會碰到 Umami 的限制。無法追蹤自訂事件或建立轉換漏斗，會成為瓶頸。

也要考慮總持有成本。自架 Umami 看起來是免費的，但您其實是在支付伺服器資源、維護時間，以及錯失洞察的成本。一旦把這些隱藏成本算進去，像 OpenPanel 這樣的雲端平台通常更有價值。

上面的匯出流程讓遷移風險更低。您可以先匯出歷史資料，再試用 OpenPanel 或其他平台，並根據實際體驗而不是功能清單來做決定。

## 資料匯出的最佳做法

匯出 Umami 資料時，時機很重要。請選擇流量較低的時段，以將對資料庫效能的影響降到最低。匯出過程是唯讀的，理論上不會影響您線上的分析，但謹慎一點總是比較好。

請務必在匯出完成後立即驗證結果。打開 CSV 檔案，確認資料看起來正確。檢查日期區間是否正確、確認所有網站都有包含在內，並檢視事件資料的格式是否正確。

如果資料量很大，可以考慮依日期區間分批匯出。這樣檔案會更容易管理，也能降低逾時問題的風險。之後如果需要，您也可以很容易地將多個 CSV 檔案合併起來。

請妥善保護您匯出的資料。這些檔案包含網站訪客的相關資訊，雖然 Umami 不會蒐集個人資料，但彙總資訊仍然可能是敏感的。如果您要長期保存或透過網路傳輸，請加密這些匯出檔。

定期匯出是很好的備份方式。即使您對 Umami 很滿意，定期匯出也能保護您免於資料遺失。您可以設定每月提醒來匯出資料，或用簡單的腳本自動化這個流程。
