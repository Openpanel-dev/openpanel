## 顶部统计

页面顶部的一排指标卡片，是快速了解项目健康状况的最快方式。每张卡片都会显示所选时间范围的数值，以及与相同长度上一周期的对比。

### 独立访客

所选周期内记录到的不同 profile ID 数量。其准确性取决于你是否使用 [identify](/zh-CN/docs/get-started/identify-users)：

- **未使用 identify**：OpenPanel 会生成一个匿名 profile ID，并每 24 小时轮换一次。若某访客在 10 个不同日期返回，会被计为 10 个独立访客，因为每天都会生成新的 ID。
- **使用 identify**：profile ID 会绑定到用户的真实身份。同一个人在 10 个不同日期访问，整个周期内只会被计为 1 个独立访客。

如果你的分析需要跨天去重，请配置 [用户识别](/zh-CN/docs/get-started/identify-users)。

### 会话数

所选周期内的会话总数。会话从有人进入站点时开始，在 30 分钟无活动后或关闭标签页时结束。一个访客在一天内可以有多个会话。

### 浏览量

在所有会话中记录到的页面浏览总数（`screen_view` 事件）。访客每加载一次页面——包括单次会话内在页面之间切换——都计为一次浏览。

### 每次会话浏览页数

单次会话内浏览的平均页面数，计算方式为 `总浏览量 / 总会话数`。数值越高，表示访客在离开前探索了更多内容。

### 跳出率

访客只浏览了单个页面就离开的会话占比。计算方式为 `单页会话数 / 总会话数 × 100`。通常越低越好——这表示更多访客不止停留在第一页。

> 如果访客在会话结束前恰好触发了 1 次 `screen_view` 事件，就会被计为一次跳出。即使访客深入阅读了一篇文章后离开，也仍然算作跳出。

### 会话时长

会话平均时长，单位为秒，仅从访客在首次页面加载后有后续行为的会话中计算（duration > 0）。访客立即离开的会话会从平均值中排除，以避免数值失真。

### 收入

所选周期内通过 `revenue` 事件追踪到的总金额，按你的账户货币显示。只有在你启用了收入事件追踪时才会显示收入。配置说明请参见 [收入追踪文档](/zh-CN/features/revenue-tracking)。

## 时间序列图表

统计卡片下方是折线图，用于展示所选指标随时间的变化。点击任意统计卡片即可将图表切换到对应指标。

图表使用你选择的 **interval**（小时、天、周或月）来聚合数据点。淡色虚线表示上一对比窗口中对应的时间段，因此你可以一眼看出趋势。

当当前指标不是 Revenue 时，图表还会在第二个 Y 轴上以绿色柱状图叠加收入数据——这样你可以在不切换卡片的情况下，将流量模式与收入相关联。

折线末尾（当前未完成的时间段）会以虚线段显示，提醒你该周期的数据仍在累积中。

## 洞察

当你的项目至少有 30 天数据后，图表下方会出现一个可滚动的洞察卡片行。OpenPanel 会自动检测页面浏览、入口页、来源和国家的显著趋势——无需任何配置。

每张卡片都会显示：
- **占比**：该属性占总流量的百分比（例如：“美国：占全部会话的 42%”）
- **绝对变化**：与上一周期相比，会话数的原始增减
- **百分比变化**：该属性相对自身上一周期值的增长或下降幅度

例如，如果美国上周有 1,000 次会话，本周有 1,200 次，那么卡片会显示“+200 次会话（+20%）”。

点击任意洞察卡片会过滤整个概览页面，只显示与该属性匹配的数据——帮助你下钻查看驱动趋势的原因。

## 来源

Sources 小组件显示访客来自哪里。可在不同标签页之间切换查看不同维度：

| 标签 | 显示内容 |
|-----|---------------|
| **Refs** | 按名称分组的 referrer（例如 “Google”、“Twitter”、“Hacker News”） |
| **Urls** | 原始 referrer URL |
| **Types** | referrer 分类：`search`、`social`、`email`、`unknown` |
| **Source** | `utm_source` 查询参数值 |
| **Medium** | `utm_medium` 查询参数值 |
| **Campaign** | `utm_campaign` 查询参数值 |
| **Term** | `utm_term` 查询参数值 |
| **Content** | `utm_content` 查询参数值 |

referrer 名称和类型会通过内置查找表，根据原始 referrer URL 自动解析。直接流量（无 referrer）会显示为 `(not set)`。

每一行都会显示会话数和浏览量。点击某一行会过滤整个概览页面，只显示来自该来源的数据。

## 页面

Pages 小组件显示访客正在进入、退出以及停留时间最长的 URL。

| 标签 | 显示内容 |
|-----|---------------|
| **Top pages** | 按独立会话数排名的页面。每一行都是一个 `origin + path` 组合。 |
| **Entry pages** | 每个会话的第一页——访客进入时所在的页面。 |
| **Exit pages** | 每个会话的最后一页——访客离开时所在的页面。 |

页面的高退出率不一定是坏事——它可能说明该页面成功回答了一个问题。入口页的高跳出率更能反映问题。对比入口页和退出页的分布，可以了解用户旅程的形态。

点击某一页面行会过滤整个概览，只显示包含该页面的会话。

## 设备

Devices 小组件按硬件和软件维度拆分你的受众。可在不同标签页之间切换：

| 标签 | 显示内容 |
|-----|---------------|
| **Device** | 设备类型：Desktop、Mobile、Tablet |
| **Brand** | 硬件品牌（Apple、Samsung 等） |
| **Model** | 具体设备型号 |
| **Browser** | 浏览器名称（Chrome、Safari、Firefox 等） |
| **Browser ver.** | 浏览器版本号 |
| **OS** | 操作系统（macOS、Windows、iOS、Android 等） |
| **OS ver.** | 操作系统版本 |

每一行都会显示会话数和浏览量。使用这个小组件来优先决定需要测试和优化哪些浏览器与操作系统。

## 事件

Events 小组件按数量展示所选周期内触发次数最多的自定义事件。系统事件（`session_start`、`session_end`、`screen_view`）会被排除——这里只显示你自行埋点的事件。

点击任意事件可过滤概览，只显示触发了该事件的会话。

## 地理位置

Geo 小组件显示访客的地理分布。可在不同标签页之间切换：

| 标签 | 显示内容 |
|-----|---------------|
| **Country** | 访客国家/地区，基于 IP 地理定位得出 |
| **Region** | 州或省 |
| **City** | 城市级别 |

表格下方的世界地图会以热力图形式展示相同数据——颜色越深表示会话越多。这可以让你快速看出受众主要集中在哪些区域。

点击国家、地区或城市会过滤整个概览，只显示该位置的数据。

## 活动热力图

页面底部的活动热力图按星期几（周一到周日）和一天中的小时（00:00–23:00）展示访客最活跃的时间。每个单元格显示所选指标在该星期和小时组合上的**平均值**，该平均值是在所选周期内按所有周求得。

颜色越深表示平均值越高。悬停任意单元格可查看精确平均值。

你可以使用热力图上方的标签切换要可视化的指标：

- **独立访客**
- **会话数**
- **浏览量**
- **跳出率**
- **每次会话浏览页数**
- **会话时长**

使用热力图识别流量高峰时段、规划活动，并在空闲时段安排维护。

## 用户旅程

页面最底部的 User Journey（Sankey）图展示访客在单次会话中如何在站点内流动。它回答了这个问题：落地到页面 A 之后，访客接下来去了哪里？

**工作方式：**

1. OpenPanel 会识别所选周期内最常见的 3 个入口页。
2. 从每个入口页开始，它会找出最常见的 3 个下一步页面（第 2 步），然后在这些页面中继续找出最常见的 3 个下一步页面（第 3 步），依此类推，直到达到配置的步数上限（默认 5 步，最多可调到 10 步）。
3. 占总会话数少于 0.25% 的路径会被过滤，以减少视觉噪声。
4. 会话中连续重复的页面会合并为一步（例如，如果有人刷新了页面，在旅程中只会计一次）。

每个节点显示页面 URL。连接流的宽度与沿该路径的会话数成正比。

使用 User Journey 查找流失点、发现意外的热门路径，并了解访客是否到达了关键转化页面。

## 过滤器和时间控件

概览页上的每个小组件都会响应页面顶部相同的一组全局过滤器和时间控件。

**Range**：选择预设值（Today、Last 7 days、Last 30 days 等）或自定义日期范围。

**Interval**：控制时间序列图表按什么粒度聚合数据（hour、day、week、month）。

**Event filter**：将整个概览缩小到包含特定事件的会话——适合分析完成某个特定动作的用户行为。

**Dimension filters**：点击任意小组件中的任意一行（国家、来源、页面）会将该值应用为过滤器。当前激活的过滤器会显示在时间控件下方的标签中。点击标签上的 × 可移除过滤器。

**Live counter**：右上角的绿色徽标显示当前活跃访客数量（在最近 5 分钟内触发过事件的访客）。点击它可查看 30 分钟会话直方图。
