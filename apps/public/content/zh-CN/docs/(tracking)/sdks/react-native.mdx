## 安装

<Steps>
### 安装依赖

我们依赖 `expo-application` 提供 `buildNumber`、`versionNumber`（以及 android 上的 `referrer`），并依赖 `expo-constants` 获取 `user-agent`。

```npm
npm install @openpanel/react-native
npx expo install expo-application expo-constants
```

### 初始化

在原生端，我们使用 `clientSecret` 认证应用。

```typescript
import { OpenPanel } from '@openpanel/react-native';

const op = new OpenPanel({
  clientId: '{YOUR_CLIENT_ID}',
  clientSecret: '{YOUR_CLIENT_SECRET}',
});
```

## 离线支持

当设备离线时，SDK 可以缓冲事件，并在恢复连接后刷新发送。事件在触发时会带上 `__timestamp`，因此即使稍后才送达，也会按正确时间记录。

要启用此功能，需要两个可选的 peer dependency：

```npm
npm install @react-native-async-storage/async-storage @react-native-community/netinfo
```

将它们传入构造函数：

```typescript
import { OpenPanel } from '@openpanel/react-native';
import AsyncStorage from '@react-native-async-storage/async-storage';
import NetInfo from '@react-native-community/netinfo';

const op = new OpenPanel({
  clientId: '{YOUR_CLIENT_ID}',
  clientSecret: '{YOUR_CLIENT_SECRET}',
  // 在应用重启之间持久化事件队列
  storage: AsyncStorage,
  // 设备重新联网时自动刷新队列
  networkInfo: NetInfo,
});
```

这两个选项彼此独立——你可以只用其中一个，也可以同时使用：

- **`storage`** — 将队列持久化到磁盘，使事件在离线期间仍能跨应用重启保留。
- **`networkInfo`** — 在连接恢复时自动刷新队列。没有它时，队列会在应用下次进入活动状态时刷新。
