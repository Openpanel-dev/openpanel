[Dokploy](https://dokploy.com) 是一个开源、自托管的应用部署平台。OpenPanel 可以通过 Docker Compose 模板部署到 Dokploy，并且有一些特定的配置要求。

<Callout type="warn">
⚠️ **重要**：Dokploy 模板需要不同于 Coolify 的特定配置。请务必仔细遵循所有步骤，尤其是环境变量和域名配置。
</Callout>
<Callout type="warn">
⚠️ **重要**：我们在 dokploy 上有一个未解决的问题 https://github.com/Dokploy/templates/issues/292，并希望它能尽快修复。
</Callout>

## 前提条件

- 已安装并运行的 Dokploy 实例
- 至少 2GB RAM 的服务器（建议 4GB+）
- 已在 Dokploy 中配置域名

## 快速开始

<Steps>
<Step>
### 部署 OpenPanel 模板

1. 登录你的 Dokploy 控制台
2. 进入你的项目
3. 点击 **"New Application"** 或 **"Deploy"**
4. 选择 **"Docker Compose"** 或搜索 **"OpenPanel"** 模板
5. 选择 OpenPanel 模板
</Step>

<Step>
### 配置域名

在 Dokploy 中配置你的域名：

1. 为控制台设置主域名（例如 `analytics.yourdomain.com`）
2. 配置 API 域名 - 这应该与控制台使用**相同域名**，并将 API 路径转发到 `/api`

<Callout>
API 和控制台使用同一个域名。API 服务有一个转发路径 `/api`，因此请确保正确配置。
</Callout>
</Step>

<Step>
### 配置环境变量

编辑 Dokploy 中的 `.env` 文件或环境变量。你**必须**设置以下环境变量：

```bash
# Required: Set these to your actual domain
API_URL=https://yourdomain.com/api
DASHBOARD_URL=https://yourdomain.com

# Database Configuration (automatically set by Dokploy)
OPENPANEL_POSTGRES_DB=openpanel-db
SERVICE_USER_POSTGRES=postgres
SERVICE_PASSWORD_POSTGRES=<auto-generated>
SERVICE_PASSWORD_REDIS=<auto-generated>

# Optional Configuration
OPENPANEL_ALLOW_REGISTRATION=false
OPENPANEL_ALLOW_INVITATION=true
RESEND_API_KEY=your-resend-api-key
OPENPANEL_EMAIL_SENDER=noreply@yourdomain.com
```

<Callout type="warn">
⚠️ **关键**：与 Coolify 不同，Dokploy 不支持 `SERVICE_FQDN_*` 变量。你**必须**使用你的实际域名值硬编码 `API_URL` 和 `DASHBOARD_URL`。
</Callout>
</Step>

<Step>
### 配置 API 服务域名设置

在 Dokploy 中，配置 API 服务域名：

1. 进入 `op-api` 服务配置
2. 设置域名配置：
   ```toml
   [[config.domains]]
   serviceName = "op-api"
   port = 3000
   host = "${api_domain}"
   ```
3. **重要**：为 API 服务勾选 **"Strip external path"** 复选框

<Callout>
"Strip external path" 选项至关重要！如果不启用，API 在接收从 `/api` 转发的请求时会得到错误路径。
</Callout>
</Step>

<Step>
### 部署

1. 检查所有配置
2. 点击 **"Deploy"** 或 **"Save"**
3. 等待所有服务启动并变为健康状态

监控部署日志，确保所有服务正确启动。
</Step>

<Step>
### 验证安装

部署完成后：

1. 检查所有服务是否运行：
   - `op-api` - API 服务器
   - `op-dashboard` - 控制台（前端）
   - `op-worker` - 后台 worker
   - `op-db` - PostgreSQL 数据库
   - `op-kv` - Redis 缓存
   - `op-ch` - ClickHouse 数据库

2. 通过你配置的域名访问控制台
3. 尝试创建账号，验证 API 是否正常工作

<Callout type="warn">
如果你在 Dokploy 前使用 Cloudflare，请记得在修改后清除 Cloudflare 缓存，以确保提供更新后的资源。
</Callout>
</Step>
</Steps>

## 配置详情

### 必需的环境变量

对于 Dokploy，你**必须**硬编码这些变量（与 Coolify 不同，Dokploy 不支持 `SERVICE_FQDN_*` 变量）：

- `API_URL` - 完整 API URL（例如 `https://analytics.example.com/api`）
- `DASHBOARD_URL` - 完整控制台 URL（例如 `https://analytics.example.com`）

Dokploy 会自动设置：
- `OPENPANEL_POSTGRES_DB` - PostgreSQL 数据库名
- `SERVICE_USER_POSTGRES` - PostgreSQL 用户名
- `SERVICE_PASSWORD_POSTGRES` - PostgreSQL 密码（自动生成）
- `SERVICE_PASSWORD_REDIS` - Redis 密码（自动生成）

要查看所有可用环境变量的完整参考，请参阅 [Environment Variables documentation](/zh-CN/docs/self-hosting/environment-variables)。

### 域名配置

API 和控制台服务共享同一个域名：

- **控制台**：在根路径（`/`）提供前端
- **API**：在 `/api` 路径提供 API

**API 服务的重要设置：**
- 域名：与控制台域名相同
- 路径：`/api`
- **Strip external path**：✅ **必须勾选**

这样可确保当请求到达 `/api/*` 时，路径会在转发到 API 服务时正确去除 `/api` 前缀。

## 故障排除

### API 请求无法工作

如果部署后 API 请求失败：

1. **验证环境变量**：
   ```bash
   # Check that API_URL is set correctly
   docker exec <op-api-container> env | grep API_URL
   docker exec <op-dashboard-container> env | grep API_URL
   ```

2. **检查 "Strip external path" 设置**：
   - 进入 Dokploy 中的 API 服务配置
   - 确保 "Strip external path" 已**勾选**

3. **验证域名配置**：
   - API 服务的路径应为 `/api`
   - 控制台服务应位于根路径 `/`

### 创建账号无法工作

如果创建账号失败：

1. 检查 API 日志：
   ```bash
   # In Dokploy, view logs for op-api service
   ```

2. 验证 `API_URL` 是否与你的域名匹配：
   - 应为 `https://yourdomain.com/api`
   - 不是 `http://localhost:3000` 或类似地址

3. 检查 API 服务是否可访问：
   ```bash
   curl https://yourdomain.com/api/healthcheck
   ```

### Cloudflare 缓存问题

如果你在 Dokploy 前使用 Cloudflare：

1. 部署或更新后，清除 Cloudflare 缓存
2. 这样可以确保立即提供更新后的资源
3. 你可以在 Cloudflare 控制台或通过 API 执行此操作

### 数据库连接问题

如果服务无法连接到数据库：

1. 验证数据库服务是否正在运行：
   - `op-db`（PostgreSQL）
   - `op-kv`（Redis）
   - `op-ch`（ClickHouse）

2. 检查环境变量是否已设置：
   ```bash
   docker exec <op-api-container> env | grep DATABASE_URL
   ```

3. 验证 docker-compose 中的服务名是否匹配：
   - 数据库服务名：`op-db`、`op-kv`、`op-ch`
   - 这些应与连接字符串中的主机名一致

## Docker Compose 结构

OpenPanel 模板包含以下服务：

- **op-api**：OpenPanel API 服务器
- **op-dashboard**：OpenPanel 控制台（前端）
- **op-worker**：用于处理事件的后台 worker
- **op-db**：PostgreSQL 数据库
- **op-kv**：Redis 缓存
- **op-ch**：ClickHouse 分析数据库

## 与 Coolify 的差异

Dokploy 模板与 Coolify 的不同点如下：

1. **环境变量**：
   - Dokploy 不支持 `SERVICE_FQDN_*` 变量
   - 必须硬编码 `API_URL` 和 `DASHBOARD_URL`

2. **域名配置**：
   - 必须手动配置域名路径
   - 必须为 API 服务启用 "Strip external path"

3. **服务发现**：
   - 使用标准 Docker Compose 服务名
   - 没有自动 FQDN 解析

## 更新 OpenPanel

在 Dokploy 中更新 OpenPanel：

1. 拉取最新镜像：
   ```bash
   docker compose pull
   ```

2. 重启服务：
   ```bash
   docker compose up -d
   ```

或者使用 Dokploy 的 UI 重启服务。
